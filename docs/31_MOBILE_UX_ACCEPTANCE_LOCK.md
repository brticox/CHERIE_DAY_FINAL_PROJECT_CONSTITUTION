# MOBILE UX ACCEPTANCE LOCK

## Mobile Navigation

Use a centered logo header with:

- menu icon on left,
- account/cart icons on right,
- compact sticky behavior,
- drawer navigation with Turkish labels,
- visible WhatsApp/contact action in drawer,
- no dense desktop nav on mobile.

## Mobile Drawer Labels

- `Deneyimler`
- `Koleksiyonlar`
- `Mağaza`
- `Dijital`
- `Hatıra`
- `Planlama`
- `Rehber`
- `İletişim`
- `Hesabım`
- `Seçimlerim`
- `Sipariş Takibi`

## Mobile Opening

Mobile opening must:

- load first meaningful visual quickly,
- show HTML headline clearly,
- simplify the approved creative concept versus desktop,
- avoid scroll trap,
- keep primary CTA visible without overlap,
- respect reduced motion,
- avoid tiny unreadable product details.

## Checkout Mobile

Checkout must:

- use one-column layout,
- show order summary collapsible,
- keep labels visible,
- use Turkish validation,
- use correct mobile keyboards,
- show legal checkboxes near final payment,
- show payment recovery path.

## Product Mobile

Product detail must show in order:

1. image/gallery,
2. name and collection/category,
3. price or quote status,
4. production time,
5. personalization/proof note,
6. CTA,
7. details,
8. related products.

## Acceptance Viewports

Test:

- 390 x 844
- 430 x 932
- 768 x 1024
- 1440 x 900
- 1920 x 1080

No text overlap, no horizontal scroll, no invisible CTA, no trapped pinned scroll.
