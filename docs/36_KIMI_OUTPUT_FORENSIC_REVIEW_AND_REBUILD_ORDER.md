# Rejected Prototype Review

The previous Kimi prototype is rejected.

This file exists only to prevent reuse of that output.
It is not a design direction.

## Why It Was Rejected

- It was a small visual prototype, not a production platform.
- It lacked real commerce architecture.
- It lacked real authentication.
- It lacked real checkout.
- It lacked real admin.
- It lacked legal consent flows.
- It used hardcoded/static product behavior.
- It did not satisfy the CHERIE DAY business requirements.

## Rule

Do not copy, polish, or continue the rejected prototype.

Build from the cleaned constitution and create any opening visual concept separately.
