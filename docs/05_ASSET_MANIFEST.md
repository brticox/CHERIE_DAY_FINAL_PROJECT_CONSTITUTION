# Asset Manifest

This manifest defines asset governance for CHERIE DAY.
It intentionally does not prescribe a specific hero/opening design.

## 1. Canonical Brand Assets

Brand source SVGs:

- `assets/brand-source/logo.svg`
- `assets/brand-source/stamp.svg`
- `assets/brand-source/logooo.svg`
- `assets/brand-source/CDD.svg`

Canonical brand-role mapping is defined in `17_BRAND_ASSET_MAP.md`.

Brand lock:

- wordmark,
- monogram,
- stamp/seal mark,
- official logo variants,
- operational QR codes if used later.

These must never be distorted, morphed, blurred, redrawn, translated, or regenerated by AI.

## 2. Creative Opening Assets

The opening experience assets will be defined after a separate creative concept is approved.

Possible asset types may include:

- still imagery,
- video,
- generated scenes,
- transparent objects,
- 3D renders,
- canvas/WebGL assets,
- posters/fallbacks,
- mobile-specific variants.

No specific object, scene, or animation is mandatory in this constitution.

## 3. Collection Assets

Final collections should each have:

- collection image/video if approved,
- palette image,
- product flat-lay or product render,
- product detail imagery,
- digital preview where relevant,
- event/styling mood image where relevant.

Rules:

- consistent warm premium grade,
- no marketplace white-box commodity feel,
- no supplier watermark,
- no unapproved third-party branding.

## 4. Product Assets

Each product should support:

- gallery images,
- thumbnail,
- material/detail closeups,
- variant imagery where relevant,
- production/proof examples where relevant,
- alt text.

## 5. Digital Preview Assets

Needed where relevant:

- digital invitation preview,
- mobile preview,
- web preview,
- QR flow preview,
- guest interaction preview,
- delivery/status preview.

Rules:

- use CHERIE DAY UI language,
- tie previews to collections/products,
- no fake third-party branding.

## 6. UI Assets

Needed:

- favicon,
- social share image,
- loading/skeleton texture if used,
- approved icon library or custom brand-safe icons,
- payment marks where legally/provider permitted.

## 7. Legal And Commerce Assets

Needed before Turkey-only commerce launch:

- invoice logo,
- email header,
- receipt/order confirmation template imagery,
- payment provider marks,
- delivery/support icons if used.

## 8. Production Rules

- Important UI text must remain editable.
- Compress raster images to WebP/AVIF where suitable.
- Compress video to web-ready MP4/WebM where suitable.
- Provide posters/fallbacks for motion assets.
- Track source, owner, status, and usage for each asset.
