# FINAL COUNCIL POLISH AND EXECUTION MANDATE

This file is the final council mandate before handing CHERIE DAY to an implementation/design agent.

## 1. Final Standard

CHERIE DAY must be built as a 21st-century Turkish luxury commerce Maison:

- creatively memorable,
- emotionally intelligent,
- commercially complete,
- operationally serious,
- mobile-perfect,
- fast,
- accessible,
- unforgettable.

The site must never look:

- plastic,
- generic,
- template-like,
- marketplace-like,
- outdated,
- overdecorated,
- slow,
- confusing,
- AI-random,
- technically impressive but commercially unclear.

## 2. Final Creative Direction

The creative opening is intentionally not fixed in this constitution.
It must be proposed separately by the creative agent and approved before implementation.

The homepage should feel like one coherent Maison journey:

`approved opening -> Maison promise -> collections -> product house -> digital/memory -> planning -> trust/footer`

No section should feel pasted under another section.

## 3. Final Store Direction

The store must be:

- beautiful enough to be remembered,
- clear enough to buy from,
- operational enough to run daily,
- complete enough to grow into a serious Turkey-first commerce platform.

Every product must clearly show:

- what it is,
- material,
- personalization,
- proof requirement,
- production time,
- delivery,
- return/cancellation rule,
- price or quote status,
- action.

## 4. Final Admin Direction

Admin is not a decorative dashboard. It is the command center.

It must support:

- products,
- prices,
- variants,
- personalization fields,
- media,
- categories,
- collections,
- inventory/made-to-order,
- orders,
- payments,
- refunds/cancellations,
- shipments,
- proof approvals,
- customers,
- support,
- legal versions,
- SEO,
- analytics,
- settings,
- audit log.

Every public commerce state needs a corresponding admin control.

## 5. Final Motion Direction

Motion must be memorable but useful:

- opening motion depends on approved concept,
- section handoffs can be choreographed,
- product browsing stays clear,
- checkout/account/legal are calm and functional,
- reduced motion is mandatory,
- mobile is simplified,
- no scroll trap.

## 6. Final QA Expectations

The implementation is not acceptable until it is checked on:

- 390 x 844 mobile,
- 430 x 932 mobile,
- 768 x 1024 tablet,
- 1440 x 900 desktop,
- 1920 x 1080 desktop.

Must verify:

- no text overlap,
- no horizontal scroll,
- opening nonblank,
- approved media/fallback loads,
- menu works,
- drawer cart works,
- product pages are clear,
- checkout is readable,
- account/order/proof routes are protected,
- admin is usable,
- RLS/security assumptions are implemented,
- accessibility basics pass,
- performance budgets are respected.

## 7. Final Council Verdict

Proceed to design and implementation only with the full constitution, production assets, and reference skills available.

Do not improvise against the constitution. Improve within it.
