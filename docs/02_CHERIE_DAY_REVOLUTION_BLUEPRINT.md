# CHERIE DAY REVOLUTION BLUEPRINT

Project: **CHERIE DAY — Wedding, Gift & Celebration Maison**

This document upgrades the whole CHERIE DAY constitution into a higher-level execution system. It focuses on product, commerce, UX, content, and operations.

This file does not prescribe the homepage/opening creative concept. The previous homepage brief has been archived outside this project. This file governs how public and internal non-homepage sections should be upgraded.

## 1. The Big Idea

CHERIE DAY should not feel like a website with pages. It should feel like a digital Maison where every section is a room in the same house.

The final experience:

> A luxury Maison that turns love, gifts, paper, digital invitations, events, and memories into one curated world.

Every page must answer:

- What world am I entering?
- What does CHERIE DAY create here?
- What can I explore?
- What can I personalize?
- What is the next elegant action?

The project must avoid generic behavior:

- no vendor directory,
- no marketplace filters,
- no commodity Shopify product grid,
- no wedding-template cliché,
- no SaaS dashboard look on public pages,
- no random animation spectacle.

## 2. Final Experience Architecture

The site is built as five connected systems:

1. **Maison Story System** — identity, philosophy, trust, process.
2. **Worlds System** — experiences and collections as complete aesthetic worlds.
3. **Commerce System** — Shop / Product House as curated Maison products.
4. **Digital + Memory System** — digital invitations, RSVP, websites, photo/film, galleries.
5. **Concierge Conversion System** — Hayalini Tasarla, quote request, WhatsApp, CRM.

Every system should cross-link to the others. No section should be isolated.

Example:

- A `Düğün` experience links to matching collections, products, digital tools, memory offering, and quote flow.
- A collection links to invitations, QR card, wedding website theme, gift box, table stationery, memory mood, and relevant experiences.
- A product links to its collection, matching pieces, digital companion, personalization, and quote/cart flow.
- A Rehber article links to shop/category, planning, digital, or quote flow depending on search intent.

## 11. Planning / Hayalini Tasarla Upgrade

`Hayalini Tasarla` is the main conversion engine and must feel like a tailor flow, not a plain form.

Flow:

1. Event type.
2. Date/season.
3. City/location.
4. Guest count.
5. Mood/palette.
6. Collection inspiration.
7. Needed modules: experience, shop/products, digital, memory, planning.
8. Budget band.
9. Upload inspiration.
10. Contact info.
11. Confirmation + WhatsApp fallback.

UX:

- one question per step,
- visible but slim progress,
- editorial microcopy,
- save partial data where possible,
- mobile-first tap targets,
- final confirmation explains next steps.

CRM result:

- create lead,
- source type `hayalini_tasarla`,
- store metadata,
- attach selected collection/products/digital/memory.

## 16. Motion System Upgrade

Motion personality:

- slow,
- tactile,
- precise,
- editorial,
- never chaotic.

Use:

- Lenis for smooth scroll,
- approved motion tooling for the opening and major story moments,
- CSS transitions for simple controls,
- Framer Motion optional for drawers/modals/page UI.

Recommended:

- controls: 150-200ms,
- drawer/menu: 280-350ms,
- cards: 250-350ms,
- section reveals: 500-700ms,
- page transitions: 500-900ms,
- opening timeline: concept-specific.

Use Burocratik-level precision, but not Burocratik visual aggression.

Avoid:

- neon,
- chaotic cursor,
- sound by default,
- WebGL shaders in MVP,
- aggressive page hijacking,
- animations that hide content.
