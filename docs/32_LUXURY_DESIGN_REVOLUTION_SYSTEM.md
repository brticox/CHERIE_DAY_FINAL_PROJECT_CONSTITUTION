# Luxury Design Operating System

This file defines design principles and operating rules.
It intentionally does not prescribe a specific hero, scene, animation, or reference-site imitation.

## Design Goal

CHERIE DAY must feel like a refined Turkish luxury maison while remaining a clear, usable commerce and service platform.

The design must support:

- trust,
- beauty,
- product desire,
- customization,
- payment confidence,
- order tracking,
- support,
- admin operations.

## Design Principles

1. Brand first, not template first.
2. Turkish language first on all public surfaces.
3. Luxury through restraint, detail, spacing, material quality, and confidence.
4. Commerce clarity must not be sacrificed for decoration.
5. Motion must clarify, guide, or delight without slowing the user.
6. Mobile must be designed intentionally, not compressed from desktop.
7. Legal, checkout, account, and admin screens must look as considered as the homepage.

## Creative Freedom

The creative opening, homepage art direction, motion concept, 2D/3D language, and generated visual world are open decisions.

They must be proposed separately as a concept package before build.

No old hero concept is binding.
No external reference is binding.
No rejected prototype is binding.

## UI Requirements

- clear navigation,
- centered or brand-appropriate logo placement as approved by design,
- visible store path,
- account and cart access,
- accessible contrast,
- consistent buttons,
- consistent form fields,
- professional product cards,
- quick view/product detail clarity,
- checkout progress visibility,
- support/inquiry access.

## Store Experience Requirements

The store must feel premium but practical:

- filters,
- categories,
- product cards,
- product detail imagery,
- production time,
- personalization options,
- proof approval status,
- delivery notes,
- return limitations,
- secure payment cues,
- related products,
- inquiry route for complex services.

## Admin Experience Requirements

The admin must feel like a modern commerce control room:

- dense but readable,
- fast scanning,
- tables with filters,
- edit drawers/forms,
- status chips,
- order timelines,
- proof review surfaces,
- payment/refund visibility,
- media/content controls,
- audit trail clarity.

## Motion Requirements

Motion may use any suitable library or method.

Required qualities:

- purposeful,
- performant,
- accessible,
- reduced-motion fallback,
- mobile-aware,
- not used as an excuse for unclear UI.

## Forbidden

- copying another site,
- copying rejected Kimi output,
- generic template landing pages,
- decorative motion that blocks commerce,
- distorted brand marks,
- cold machine-written Turkish,
- admin screens that feel unfinished,
- checkout that feels less premium than the homepage.
